# UI仕様（画面構成・コンポーネント）

画面一覧・遷移は [feature/overview.md](../feature/overview.md)、要件は [requirements.md 第6章](../requirement/requirements.md) を参照。
ルーティングは React Router の HashRouter（GitHub Pages の404回避）。

## ルーティング

| パス | 画面 | コンポーネント |
|------|------|---------------|
| `#/` | SCR-01 ダッシュボード | `DashboardPage` |
| `#/practice/setup` | SCR-02 演習設定 | `PracticeSetupPage` |
| `#/practice/quiz` | SCR-03 問題演習 | `QuizPage` |
| `#/practice/result` | SCR-04 結果・解説 ★ | `ResultPage` |
| `#/bookmarks` | SCR-05 ブックマーク | `BookmarksPage` |
| `#/progress` | SCR-06 進捗分析 | `ProgressPage` |
| `#/admin` | SCR-07 管理 | `AdminPage` |

共通レイアウト: ヘッダー（アプリ名・ナビ）＋メインコンテンツ。演習中（SCR-03）はナビを最小化しタイマー・進捗を表示。

## SCR-02 演習設定

- モード選択（ミニ/1問1答/模試）→ モードに応じて分野（複数可）・問題数（10/20）の選択UIを出し分け
- 模試は「65問・130分・図はヒント表示不可」の説明を出して即開始
- セッション単位の図表示タイミング上書き（FR-17）

## SCR-03 問題演習

```
┌────────────────────────────────────────┐
│ 進捗 n/N   ⏱ 残り時間(模試)   [見直しフラグ]│
├────────────────────────────────────────┤
│ 問題文（stem）                           │
│ （設定が「出題時」なら DiagramView hint）  │
│ ○/☐ 選択肢（single=ラジオ / multiple=   │
│   チェック＋「N個選択してください」）       │
│              [回答する]                  │
└────────────────────────────────────────┘
```

- 模試のみ: 問題一覧ドロワー（未回答・フラグ付きへジャンプ）、残り10分でタイマー警告色
- 回答ボタンは選択が有効になるまで disabled

## SCR-04 結果・解説 ★コア画面

```
┌────────────────────────────────────────┐
│ （セット演習/模試）得点率・合否・分野別内訳 │
│ 各問正誤一覧（クリックで該当問題の解説へ）  │
├──────────────┬─────────────────────────┤
│              │ 選択肢別 ◯✗ と理由(FR-08) │
│ DiagramView  │  - <ref>語句</ref> は     │
│ （中央・大） │    ホバー/タップで図と連動  │
│  パン/ズーム  │ 解説サマリ（refIds連動）   │
│  [拡大表示]  │ 公式リンク(FR-09)         │
│              │ [ブックマーク☆] [次の問題] │
└──────────────┴─────────────────────────┘
```

- 図と解説の双方向ハイライトの詳細は [feature/diagram_highlight.md](../feature/diagram_highlight.md)
- モバイル幅（<768px）では図を上、解説を下の縦積みに切り替え
- 1問1答モードでは一覧を省略し、単問の解説＋[次の問題]

## SCR-05 ブックマーク

- 要復習問題のリスト（問題文冒頭・分野バッジ・直近正誤）
- 絞り込み: 分野／直近の正誤
- アクション: 個別に解く／全件で演習開始／ブックマーク解除

## SCR-06 進捗分析

- 分野別正解率（レーダー＋バー、未演習は「未演習」表示）
- 重点復習の提示（最低2分野）→ ミニテストへの導線
- 模試スコア推移（折れ線＋合格ライン基準線）、過去模試の再閲覧

## SCR-07 管理

- タブ構成: 「JSONインポート検証」「問題編集」「学習履歴バックアップ」
- インポート検証: テキストエリア/ファイル選択 → 検証結果一覧（OK件数・エラー箇所） → DiagramSpecプレビュー → 検証済みJSONダウンロード
- バックアップ: 学習履歴のエクスポート／インポート（FR-18）

## 共通UI原則

- Tailwind CSS のデザイントークン（spacing/色）に統一。ダークモードは Phase 4 検討
- 対話部品（ダイアログ・タブ・ラジオ等）は Radix UI を用い、キーボード操作・フォーカス管理を担保（NFR-07）
- 正誤・警告は色＋アイコン＋テキストで冗長に伝える（色のみに依存しない）
- 分野は一貫した色とラベルで表示（例: SECURE_ARCH=青 / RESILIENT=緑 / HIGH_PERF=橙 / COST_OPT=紫）
